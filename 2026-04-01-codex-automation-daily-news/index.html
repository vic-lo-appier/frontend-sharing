什麼是 Codex Automation？

核心定義
把一段會做事的 prompt，變成固定會執行的 agent workflow。
你可以指定
  • 什麼時候跑
  • 在哪個 workspace 跑
  • 使用哪個 model
  • 跑完輸出什麼
對工程流程的價值
  • 取代一次性 chat 操作
  • 不只跑 script，還能帶 reasoning
  • 適合 digest、triage、summary 類工作
官方來源與限制
官方文件： developers.openai.com/codex/app/automations
目前這個 automation 功能只有 Codex desktop app 能使用。

Automation 有哪些功能？

核心功能
  • 定時執行 prompt
  • 指定工作目錄與上下文
  • 指定 model / reasoning effort
  • 把 skill 當成工作流程入口
  • 產出檔案、寄信、開 inbox item
在晨報系統裡
automation 負責每天觸發，真正的內容生成與交付則交給 skill 和 script。
一句話理解
automation 決定「什麼時候做」，skill 決定「怎麼做」。目前這套體驗是建在 Codex desktop app 上。

Skill 內容長什麼樣？

Brief skill 會定義
  • 追哪些來源
  • 怎麼排序與去重
  • 輸出結構與欄位
  • 交付成 HTML 電子報
AI / Frontend 差異
同樣都是晨報，但各自有不同 sources、優先主題、工程判準與語氣。
Helper skill 會定義
  • 如何把 briefing 轉成 HTML
  • 如何寄送 multipart email
  • SMTP 需要哪些 env vars
設計取向
prompt 保持短，能力放進 skill；最後的 artifact 直接就是可預覽、可寄送的 HTML。
workflow 不是藏在 prompt，而是放在 skill5 / 12

Skill 實際內容範例

以 `ai-engineer-brief` 為例
ai-engineer-brief/
├── SKILL.md
├── references/
│   └── sources.md
└── agents/
    └── openai.yaml
`SKILL.md` 裡面放什麼
  • Defaults: 時間範圍、候選數量
  • Workflow: 收集、排序、去重、薄新聞處理
  • Output Format: Today / X Watch / One-Line Take / Worth Trying
  • Delivery: 產出 HTML、寄信、輸出限制
`sources.md` 裡面放什麼
  • 官方來源清單
  • 追蹤的 X / Twitter 帳號
  • Watch keywords
  • Selection heuristics
`agents/openai.yaml` 的角色
主要提供 UI 顯示名稱、簡短描述、預設 prompt，讓這個 skill 在實際使用時更容易被發現與呼叫。
把 skill 想成 prompt、references、UI metadata 的組合6 / 12

render_brief_email.py 做什麼？

功能摘要
  • 讀入 briefing 結構
  • 解析 Today / X Watch / One-Line Take / Worth Trying
  • 依標題自動切 AI / Frontend theme
  • 輸出最後的 *-email.html
主流程
markdown_file
  ↓
split_sections()
  ↓
parse_cards() / parse_paragraphs()
  ↓
render_html()
  ↓
write *-email.html
這支 script 的價值
把 briefing 內容穩定轉成可直接閱讀、可直接寄送的 HTML 電子報，讓產出格式不需要每次都靠 prompt 臨場組裝。

Automation 的利與弊

優點
  • 不用做 UI / backend / database
  • 排程、內容生成、寄信一次串起來
  • 很適合 digest / triage / summary 類工作
  • 維護成本比做完整產品低很多
限制
  • 不適合大量 end users 的正式產品
  • 不適合複雜權限模型與穩定公開 API
  • 對 prompt / skill 設計品質有依賴
  • 仍需管控來源品質與寄信設定
這是 workflow automation，不是 SaaS replacement10 / 12

Claude、Cursor 也能做到類似的事

Claude Code
  • 有 Scheduled Tasks，可定時執行 Claude Code 工作
  • 適合把固定時間會跑的 agent workflow 接到日常流程裡
  • 模式上很接近這次晨報系統的自動化需求
Cursor
  • 目前是 Automations 搭配 Background Agent
  • 可讓 agent 在雲端環境持續執行指定工作
  • 比較像把 agent execution 與排程能力整合進產品裡
官方來源
Claude Code Scheduled Tasks: code.claude.com/docs/en/scheduled-tasks
Cursor Automations: cursor.com/docs/cloud-agent/automations
類似模式在其他 agent coding 工具也已經成立11 / 12
