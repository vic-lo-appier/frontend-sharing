#1
★★★★★
OpenAI 把 agent 風險正式納入 Safety Bug Bounty
2026-03-25
Introducing the OpenAI Safety Bug Bounty program
摘要
OpenAI 在既有 security bounty 之外新增 Safety Bug Bounty，明確接受 AI abuse 與 safety risk 回報，範圍直接點名 agentic risks、第三方 prompt injection、資料外洩與 MCP 相關場景。
Why it matters
這代表 agent 產品的風險模型開始從傳統資安擴展到工具鏈、提示注入與代理越權，內部 red team 與外部 bounty 類別都該跟著調整。
Analyst這是立即可行動的更新，做 agent 平台或 MCP 整合的團隊現在就該重寫測試清單。
#2
★★★★★
OpenAI 公開內部 coding agent 監控做法
2026-03-19
How we monitor internal coding agents for misalignment
摘要
OpenAI 說明其內部 coding agent 監控系統如何以 GPT-5.4 Thinking 高推理強度檢查完整對話、工具呼叫與 chain-of-thought，並在 30 分鐘內完成低延遲審查。文中也揭露其已監控數千萬條 agentic coding 軌跡，並以控制評估測量對高風險行為的攔截能力。
Why it matters
如果你正把 agent 放進真實 repo、CI 或內部系統，監控不再只是 logs 與審計軌跡，而是要能判讀行為與意圖是否偏離任務。
Analyst這篇更像業界 baseline 草案，會影響企業如何設計 agent observability 與人審流程。

Cursor 讓 cloud agents 可自架在企業內網
2026-03-25
Run cloud agents in your own infrastructure
摘要
Cursor 宣布 self-hosted cloud agents 正式 GA，強調 codebase、tool execution 與 build artifacts 都可留在企業自己的網路環境內，同時保留 isolated remote environments、多模型、plugins 與 team permissions。

Analyst這是 enterprise agent 落地的重要拼圖，特別是已有私有 cache、內部 endpoint 與受管制憑證的團隊。

2026-03-19
Introducing Composer 2
摘要
Cursor 發表 Composer 2，宣稱在 CursorBench、Terminal-Bench 2.0 與 SWE-bench Multilingual 都有顯著提升，並把定價壓到每百萬 input token 0.50 美元、output token 2.50 美元，另有更快的 fast 變體。

Analyst這不是單純的新模型消息，而是 agent 經濟性開始可與真人流程正面競爭。

One-Line Take

AI 工程正在從「哪個模型更強」轉向「哪個 agent 工作流更可管、可驗證、可在真實企業環境中落地」。

Worth Trying
  • 把你們現有的 agent red-team 清單補上第三方 prompt injection、MCP data exfiltration、越權工具呼叫與長回合 sabotage 測項。
  • 針對一條真實內部開發流程建立 agent observability 樣板，至少記錄任務意圖、工具軌跡、關鍵推理判讀與人工介入點。
  • 選一個 legacy code modernization 或內網 CI 任務，對照自架 agent 與雲端 agent 的權限、快取、網路存取與成本差異做一天內的小規模試點。
